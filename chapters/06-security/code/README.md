# Chapter 6: 動かし方

**学習目的のみ**。生成される HTML には学習用の `<script>` が含まれます。コードを読んで何が起きるか理解してからブラウザで開いてください。**他人のサービスに対しては絶対に試さない**こと。

## SQL インジェクション

```bash
cd chapters/06-security/code

# 脆弱版: 攻撃文字列 ' OR '1'='1 でログインが通ってしまう
python 01_sqli_vulnerable.py

# 修正版: 同じ攻撃文字列を渡しても通らない
python 01_sqli_fixed.py
```

両方とも標準ライブラリ (`sqlite3`) だけで動きます。

## XSS

```bash
# 脆弱版: vulnerable.html を生成。ブラウザで開くと alert がポップアップ
python 02_xss_vulnerable.py

# 修正版: fixed.html を生成。<script> が文字列として表示されるだけ
python 02_xss_fixed.py
```

生成された HTML はファイルとしてブラウザで開けます（ダブルクリックでも `file:///...` でも可）。

## 体験のキモ

「脆弱版で攻撃が成立する」→「修正版で同じ攻撃が通らなくなる」を **自分の目で 2 回ずつ確認** すること。実物を見ることで「なぜプレースホルダを使うのか」「なぜエスケープするのか」が腑に落ちます。
